# 0.16.0

Recommended release type: **minor**.

This release should be cut as `0.16.0`, not `0.15.1`. The diff from the last
release tag, `v0.15.0` from 2026-04-09, is not a patch-sized maintenance
change. It introduces new native app surfaces, iOS and Linux targets, a new
Zig runtime layer, release automation, and broad behavior changes across auth,
terminal sessions, live-run inspection, and testing.

Note: `v0.15.0` is not an ancestor of the current branch, so this changelog is
based on a snapshot comparison of `v0.15.0..HEAD` rather than a strictly linear
commit range.

---

## Highlights

- Reworked the repository around the native TABMONSTERS app, with macOS
  SwiftUI surfaces for dashboards, runs, live run inspection, agents,
  approvals, tickets, workflows, issues, landings, scores, memory, search,
  logs, diffs, prompts, triggers, SQL browsing, and workspace management.
- Added cross-platform product work for iOS and Linux:
  - iOS app target with OAuth sign-in, onboarding, settings, workspace/session
    flows, share extension wiring, TestFlight build support, privacy manifests,
    and a Ghostty-backed terminal renderer.
  - Linux GTK/libadwaita shell with dashboard, runs, workflow, approval,
    ticket, VCS, scores, memory, live-run, DevTools, chat, search, and palette
    parity work.
- Added `libsmithers`, a Zig runtime core that now owns session, terminal,
  persistence, DevTools, transport, workspace, and FFI responsibilities used by
  the native clients.
- Added native terminal/session infrastructure with Ghostty integration,
  persisted sessions, reconnect behavior, scrollback replay, tmux-style
  reattach, bundled session helper binaries, terminal restart, and shortcut
  footer support.
- Added live-run DevTools support with snapshots, streams, event deltas,
  metrics/events tabs, heartbeat surfacing, reconnect recovery, gap resync,
  ghost-node retention, frame scrubbing, tree expansion persistence, and run
  tab lifecycle improvements.
- Rebranded Plue-facing names to Smithers across environment variables,
  scripts, CI, UI labels, runtime defaults, and documentation.

## Runtime And Platform

- Added `libsmithers` session daemon and connect helper binaries, plus Swift
  wrappers and native app resource bundling.
- Added Zig FFI APIs, C headers, module maps, Swift runtime wrappers, and
  concurrency hardening around FFI handles.
- Added runtime-backed action resolution, shape/event data-plane wiring,
  production event pumping, write acknowledgements, PTY byte propagation, and
  Swift/Zig schema alignment.
- Added Electric, WebSocket PTY, HTTP transport, SQLite, Ghostty VT, and
  Swift/Zig proof-of-concept work that fed the production runtime.
- Added observability primitives in `libsmithers`: ring buffers, histograms,
  entrypoint timing, DevTelemetry recording, and stores for metrics/events.
- Added concurrent-safe SQLite persistence, handle lifetime hardening,
  refcounting, closed flags, embedded-NUL rejection in cwd resolution, and
  typed fallback payload preservation.
- Added Android core canary and additional mobile/runtime proof-of-concept
  projects for future platform work.

## Migration Notes

- Release consumers should expect new feature surfaces and behavior changes,
  not only bug fixes. This is why the release should be `0.16.0`.
- Builds now require initialized submodules and Zig `0.15.2`; Ghostty must be
  built once with `zig build ghostty` when the xcframework is missing.
- macOS release builds require Developer ID signing credentials and Apple
  notarization environment variables.
- iOS/TestFlight builds now depend on the generated `libsmithers` and
  Ghostty-related xcframeworks, privacy manifests, and production base URL
  configuration.
- Plue-prefixed environment variables, labels, and paths have generally moved
  to Smithers naming.
