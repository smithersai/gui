# Implementation Plan — 0187 iOS Sign-Out Runtime Cache Wipe

## Scope
- Add an iOS-owned production `SessionWipeHandler` coordinator that centralizes local sign-out wipe behavior.
- Ensure `TokenManager.localSignOut()`/`signOut()` paths invoke this coordinator in production wiring.
- Wipe runtime/session/cache surfaces on sign-out or refresh lockout:
  - stop active runtime-backed terminal transport/session lifecycle
  - clear runtime cache directories used by iOS runtime host
  - clear user-scoped in-memory iOS shell state (opened workspace, focused workspace/approval/deeplink state, switcher rows/session state)
- Preserve E2E synthetic token mode behavior (no regression to env-seeded sign-in flow).
- Add focused unit tests for handler invocation and iOS state/cache wipe effects.

## Files To Change
- New: `ios/Sources/SmithersiOS/IOSSessionWipeCoordinator.swift`
  - Define iOS coordinator implementing `SessionWipeHandler`.
  - Own runtime transport shutdown + runtime cache directory wipe + broadcast/reset hooks for shell-scoped state.
- Update: `ios/Sources/SmithersiOS/SmithersApp.swift`
  - Install coordinator as `TokenManager.wipeHandler` in production app wiring.
  - Keep E2E token-store setup intact.
- Update: `ios/Sources/SmithersiOS/ContentShell.iOS.swift` (injection/wiring only)
  - Inject coordinator/state-reset hooks so shell can register/unregister runtime/session reset participants.
  - Ensure shell user-scoped state reset path is reachable from wipe coordinator.
- New/Update tests under `ios/Tests/SmithersiOSTests`:
  - `TokenManager` wipe-handler invocation test (or extend existing auth tests).
  - Coordinator tests for runtime/session host + in-memory shell reset behavior.
  - Switcher stale-row regression test across synthetic user transition.
  - Terminal-active sign-out coverage (automated preferred; otherwise document manual notes).

## Sequencing
1. Inventory/reset contract
- Enumerate concrete iOS resettable surfaces (runtime host, terminal transport owner, switcher VM rows/loading state, opened/focused shell state, deeplink focus if user-scoped).
- Define a minimal coordinator API for registering reset participants and performing idempotent wipe.

2. Build coordinator
- Implement `IOSSessionWipeCoordinator` as the single iOS `SessionWipeHandler` owner.
- Implement ordered wipe:
  1) local runtime/session shutdown + in-memory state reset
  2) runtime cache directory removal/reset
  3) return (network revocation stays in `TokenManager.signOut()` after local invalidate)

3. Wire production app
- In `SmithersApp.makeAuthModel`, attach coordinator to the production `TokenManager` instance.
- Keep synthetic/E2E mode branch working (same manager semantics, no extra auth gate changes).

4. Wire shell injection
- In `IOSContentShell`, inject coordinator participation hooks only (no unrelated UI refactors).
- Register runtime/session host + shell state reset closures and ensure lifecycle-safe unregister/deinit behavior.

5. Add tests
- `TokenManager.localSignOut()` triggers wipe handler.
- Coordinator wipe clears runtime/session host state and shell-scoped state.
- Sign-out/sign-in as another synthetic user does not show stale switcher rows.
- Terminal-active sign-out path stops transport and leaves no stale mounted terminal session.

6. Validate and stabilize
- Run Swift package + iOS scheme tests and fix race/isolation issues.

## Validation Strategy
- Automated:
  - `cd Shared && swift test`
  - `xcodebuild -project SmithersGUI.xcodeproj -scheme SmithersiOS -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.6' -configuration Debug CODE_SIGNING_ALLOWED=NO test`
- Assertions to include:
  - `TokenManager.localSignOut()` calls `wipeAfterSignOut()` exactly once.
  - Coordinator wipe is idempotent and safe when no active runtime session exists.
  - Runtime transport/session is stopped before cache wipe is asserted.
  - Post-wipe shell models expose empty/default user-scoped state.
- Manual fallback (if transport test remains flaky under simulator):
  - Sign in, open workspace terminal, trigger sign-out, verify terminal detaches and next user sees clean switcher/detail state.

## Known Risks
- Concurrency/isolation risk between `@MainActor` iOS shell state and `TokenManager` sign-out call-sites; coordinator should enforce main-actor mutation for UI-bound state.
- Lifecycle coupling risk if coordinator retains view models strongly; use weak/ephemeral registrations to avoid leaks.
- Cache wipe timing risk if runtime still holds file handles; stop transports/sessions before removing cache directory.
- Cross-ticket merge risk with 0186 in `SmithersApp.swift`; keep patch minimal and coordinate ordering of auth wiring edits.
- E2E mode regression risk if coordinator wiring accidentally changes synthetic token bootstrap assumptions.
