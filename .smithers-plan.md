# Implementation Plan — Ticket 0186 iOS Authenticated Request Adoption

## Scope
- Migrate owned iOS feature HTTP paths from raw bearer + direct `URLSession` usage to the shared refresh-aware authenticated HTTP client introduced by ticket 0184.
- Cover approvals list/decision, workflow runs list/cancel/rerun/resume, repo selection requests, and workspace create/delete/suspend/resume/snapshot/fork actions in owned files.
- Preserve existing UI-facing error semantics: `429 Retry-After`, quota, missing repo context, backend-unavailable, and signed-out/auth-expired mapping.
- Keep E2E environment behavior unchanged.
- Do not modify `Shared/Sources/SmithersAuth` except consuming the 0184 public API.

## Sequencing
1. Baseline audit in owned source files:
- Identify each raw token/read path (`try? tokenManager.currentAccessToken()` or equivalent), direct `URLSession` call site, and local HTTP error mapping.
- Record required request/response behavior per endpoint to preserve UI state transitions.

2. Introduce authenticated-client adoption per feature surface:
- Replace approvals networking with 0184 authenticated client.
- Replace workflow runs networking with 0184 authenticated client.
- Replace repo/workspace action networking with 0184 authenticated client.
- Ensure each call path converts shared-client auth-expired outcome into existing signed-out/auth-expired UI state.

3. Preserve domain-specific error behavior:
- Keep existing handling for `429 Retry-After` (including surfaced retry timing/message behavior).
- Keep quota/missing repo context/backend unavailable mappings unchanged.
- Ensure refresh failure/auth failure is surfaced (not swallowed) and routed to expected UI state.

4. Test updates and additions:
- Add/adjust tests proving approvals list + decision retry once after synthetic `401`.
- Add/adjust tests proving workflow runs list/cancel/rerun/resume retry once after synthetic `401`.
- Add/adjust tests proving repo/workspace actions preserve `429 Retry-After` behavior.
- Add/adjust tests proving missing bearer maps to auth-expired/signed-out.
- Add assertions that owned production files no longer use raw bearer fetch pattern (except test fixtures).

5. Final verification and cleanup:
- Run shared tests and iOS scheme tests.
- Spot-check E2E env-dependent code paths in modified files to confirm no behavior regressions.

## Validation Strategy
- Automated:
- `cd Shared && swift test`
- `xcodebuild -project SmithersGUI.xcodeproj -scheme SmithersiOS -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.6' -configuration Debug CODE_SIGNING_ALLOWED=NO test`
- Focused assertions:
- Single retry on synthetic `401` for approvals/workflow-runs operations.
- `429 Retry-After` propagation unchanged for repo/workspace actions.
- Missing bearer -> signed-out/auth-expired mapping retained.
- Static grep check in owned source files for prohibited raw token pattern (allowing fixtures/tests).

## Known Risks
- Ticket 0184 API shape/behavior may still be in flux; integration points may need small adaptation once finalized.
- Error mapping regressions are easy when consolidating transport layer; especially `429` and backend-unavailable edge cases.
- Retry semantics could accidentally become 0x or >1x if shared client defaults differ from current per-feature behavior.
- E2E env support could regress if request construction paths currently carry environment-specific headers/URLs outside shared client wrappers.
