# Implementation Plan — Ticket 0188 iOS Production Terminal Attach

## Scope
- Decouple production iOS terminal attach from seeded E2E environment variables and seeded-only routing assumptions.
- Drive terminal probing/attach from selected workspace and backend-provided session context.
- Standardize terminal WebSocket URL resolution: prefer backend-provided URL, otherwise use one centralized constructor with tests.
- Ensure user-visible terminal states cover: no session yet, checking session, session missing, attach failed, terminal active, and kill-switch disabled.
- Preserve seeded env-var attach path strictly as an explicit E2E/testing shortcut.

## Files To Change
- `ios/Sources/SmithersiOS/ContentShell.iOS.swift`
  - Move production attach entrypoint to workspace-selected flow.
  - Gate seeded env usage behind explicit E2E/test-only branch.
- `Shared/Sources/SmithersStore/WorkspaceSessionPresenceProbe.swift`
  - Update probe contract to consume workspace/repo context and backend response fields needed for attach.
  - Emit explicit state model values required by UI.
- `ios/Sources/SmithersiOS/Terminal/TerminalSurface.swift` (only if needed for attach status/reporting)
  - Add/align terminal state reporting for attach lifecycle and failure reasons.
- `libsmithers/src/core/transport.zig` (conditional)
  - If no backend URL is provided, centralize fallback WS URL construction and keep it as single source of truth.
- Tests
  - `ios/Tests/SmithersiOSTests/WorkspaceSessionPresenceProbeViewTests.swift`
  - `ios/Tests/SmithersiOSTests/TerminalSurfaceConnectionStateTests.swift`
  - `ios/Tests/SmithersiOSTests/WorkspaceDetailActionsTests.swift` (or nearest workspace-detail flow tests)
  - `ios/Tests/SmithersiOSE2ETests/SmithersiOSE2ETerminalTests.swift`
  - `ios/Tests/SmithersiOSE2ETests/SmithersiOSE2ETerminalExtendedTests.swift` (if seeded scenario currently lives here)
  - `libsmithers` transport/core tests for URL contract behavior.

## Sequencing
1. Baseline and inspect current attach path
- Trace production attach path from workspace selection to probe to `TerminalSurface` mount.
- Isolate all reads of `PLUE_E2E_WORKSPACE_SESSION_ID` and identify current fallback route builder(s).

2. Refactor probe and attach inputs
- Update `WorkspaceSessionPresenceProbe` to resolve session/attach metadata from workspace selection + backend response.
- Ensure probe output differentiates `noSessionYet`, `checking`, `sessionMissing`, `readyToAttach`, and `attachFailed`-adjacent signals.

3. Update iOS shell wiring
- In `ContentShell.iOS.swift`, make production default use probe/workspace context only.
- Keep seeded env var flow as explicit E2E shortcut branch (opt-in, not default).

4. Normalize terminal URL contract
- Implement “backend URL first” attach behavior.
- If fallback is needed, move URL construction to centralized helper (and `transport.zig` only if shared runtime contract requires it).
- Document contract inline at single source location and reference it from tests.

5. UI state rendering polish
- Ensure workspace detail / terminal panel presents all required states clearly.
- Keep kill-switch-disabled state distinct from generic attach failure.

6. Tests
- Add/adjust unit tests for workspace with no session => production CTA/empty state.
- Add/adjust unit tests for workspace with session => `TerminalSurface` mounts.
- Add unit tests for URL contract precedence (backend URL vs fallback constructor).
- Keep E2E seeded env test path passing as explicit shortcut.

7. Verification and regression pass
- Run iOS and libsmithers test suites from ticket.
- Confirm acceptance criteria mapping in test names/assertions.

## Validation Strategy
- Primary commands:
  - `xcodebuild -project SmithersGUI.xcodeproj -scheme SmithersiOS -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.6' -configuration Debug CODE_SIGNING_ALLOWED=NO test`
  - `cd libsmithers && zig build test --summary all`
- Targeted checks before full suites:
  - Run affected iOS unit test files for probe/state and terminal mount behavior.
  - Run terminal E2E test subset to confirm seeded shortcut still works.
- Assertion focus:
  - Production path never requires `PLUE_E2E_WORKSPACE_SESSION_ID`.
  - State transitions are deterministic and user-visible for all required states.
  - URL resolution precedence is stable and covered by unit tests.

## Known Risks
- Hidden coupling to ticket 0185 PTY lifetime behavior; avoid lifetime semantics changes in this ticket.
- URL-contract drift risk between iOS and core transport if fallback logic is duplicated; mitigate with single constructor and tests.
- Regressions in E2E seeded scenarios if shortcut branch is not clearly isolated from production path.
- Backend response variability (missing session/url fields) may surface new edge states; handle with explicit state mapping instead of implicit failure.
