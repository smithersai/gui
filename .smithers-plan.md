# Ticket 0193 Implementation Plan — CI release + XcodeGen hardening

## Scope
- Harden iOS simulator selection in CI so builds/tests use an available runtime/device on each runner.
- Make XcodeGen drift check deterministic by running from canonical repo root and validating only committed project output.
- Update TestFlight script/workflow behavior so `SKIP_UPLOAD=1` does not require App Store Connect API key secrets.
- Add explicit signing/profile preflight checks before archive/export in release path.
- Retain private release diagnostics (`.ipa`, export plist, dSYMs) as workflow artifacts.
- Align release documentation with script and workflow behavior, including local dry-run commands and required env vars.

## Files To Change
- `.github/workflows/ci.yml`
- `.github/workflows/ios-testflight.yml`
- `ios/scripts/build-and-upload-testflight.sh`
- `ios/RELEASE.md`
- `project.yml` only if a build setting change is required for signing/export behavior (otherwise no change)

## Sequencing
1. Baseline and branch-safe command hardening
- Confirm current CI/TestFlight commands and assumptions in the four target files.
- Define a single simulator resolution approach for CI:
  - Prefer runtime device UUID discovery via `xcrun simctl list --json` + selection logic.
  - Fallback to ordered `-destination` candidates (e.g., newest iPhone available) if UUID resolution fails.

2. CI workflow updates (`.github/workflows/ci.yml`)
- Add a step that resolves an available iPhone simulator destination at runtime and exports it (job env/output).
- Replace hardcoded simulator destination in iOS build/test steps with resolved destination.
- Ensure xcodegen drift check runs from repo root deterministically:
  - Explicit `working-directory` or `cd "$GITHUB_WORKSPACE"` before `xcodegen generate` and diff check.
  - Keep diff scope constrained to `SmithersGUI.xcodeproj`.

3. Release script hardening (`ios/scripts/build-and-upload-testflight.sh`)
- Gate required env checks by execution mode:
  - Always require signing/profile inputs needed for archive/export.
  - Only require `APP_STORE_CONNECT_API_KEY_*` when upload is enabled.
- Add preflight validation before archive:
  - Validate required signing vars are non-empty.
  - Validate provisioning profiles can be located/decoded for expected bundle IDs where feasible.
  - Validate `xcodegen`/tool prerequisites with actionable error messages.
- Preserve existing exit-code contract, extending only if necessary and documented.

4. TestFlight workflow updates (`.github/workflows/ios-testflight.yml`)
- Keep upload behavior unchanged for normal runs, but ensure dry-run path (`SKIP_UPLOAD=1`) is supported without ASC secrets.
- Upload additional private diagnostics on `always()`:
  - `build/ios-archive/*.dSYM` and/or dSYMs inside `*.xcarchive`.
  - Existing `.ipa` + `ExportOptions.plist` artifact retention remains.
- Use limited retention window and no public exposure (GitHub private artifacts only).

5. Documentation alignment (`ios/RELEASE.md`)
- Update env var requirements to distinguish:
  - Archive/export required vars.
  - Upload-only vars.
- Add/refresh local dry-run section:
  - `SKIP_UPLOAD=1 ./ios/scripts/build-and-upload-testflight.sh`
  - Expected prerequisites and what is intentionally skipped.
- Reflect artifact behavior and troubleshooting guidance for preflight failures.

6. Optional build-setting adjustment (`project.yml`)
- Only if needed to fix signing/export determinism discovered during validation.
- Regenerate with `xcodegen generate` and verify project drift scope remains expected.

## Validation Strategy
- Deterministic generation and drift checks:
  - `xcodegen generate`
  - `git diff --quiet -- SmithersGUI.xcodeproj`
- Simulator resilience check:
  - Run updated CI destination-resolution logic locally (or via shell extraction) to ensure it selects an available simulator.
  - Execute iOS simulator build using resolved destination.
- Release dry-run verification:
  - `SKIP_UPLOAD=1 ./ios/scripts/build-and-upload-testflight.sh`
  - Confirm no requirement for `APP_STORE_CONNECT_API_KEY_ID`, `APP_STORE_CONNECT_ISSUER_ID`, `APP_STORE_CONNECT_API_KEY_P8` in dry-run mode.
- Release workflow artifact verification:
  - Inspect workflow artifact paths for `.ipa`, `ExportOptions.plist`, and dSYM payload.
- Documentation verification:
  - Ensure `ios/RELEASE.md` env var matrix and commands match script behavior exactly.

## Known Risks
- Runner image differences may still affect simulator naming/runtime availability; UUID discovery and fallback ordering mitigate but do not eliminate all edge cases.
- dSYM artifact size may increase job upload time/storage; retention-days should remain constrained.
- Preflight strictness can initially fail previously “working by accident” setups (profile naming mismatches, missing extension profile).
- Conditional secret requirements must not weaken normal upload path validation.
- If project.yml changes become necessary, generated `.xcodeproj` churn can obscure unrelated diffs; keep drift scope tight and review stat output.
