# 0185 Implementation Plan — Runtime PTY Lifetime and Concurrency Hardening

## Scope
- Harden Swift↔Zig PTY/session lifetime so PTY wrappers cannot outlive valid session state unsafely.
- Ensure disconnect/detach semantics are safe when PTY handles are still referenced.
- Eliminate worker publication races in Zig transport (subscription and PTY workers).
- Eliminate write/resize vs detach/destroy use-after-free windows via explicit ownership/lifetime discipline.
- Keep public API shape stable where practical; allow internal initializer/lifetime plumbing changes.
- Explicitly out of scope: iOS/UI terminal behavior (owned by ticket 0186).

## Files To Change
- `Shared/Sources/SmithersRuntime/SmithersRuntime.swift`
- `Shared/Tests/SmithersRuntimeTests/SmithersRuntimeTests.swift`
- `libsmithers/src/core/ffi.zig`
- `libsmithers/src/core/transport.zig`
- `libsmithers/test/**` only if additional Zig integration/stress harness coverage is needed beyond `transport.zig` + `ffi.zig` unit tests.

## Sequencing
1. **Lock down target invariants (no API drift first):**
- Document/encode invariants in `ffi.zig` + `transport.zig` comments/tests before refactors:
  - Session remains alive while any PTY box still owns a strong ref.
  - PTY operations fail cleanly once detached/closed, never UAF.
  - Worker entries are not published before `thread` (and required lifetime fields) are initialized.

2. **Swift lifetime ownership hardening (`SmithersRuntime.swift`):**
- Make PTY wrapper ownership explicit and one-way safe until detach/deinit completion.
- Move any detach state mutation to an atomic/locked one-shot path so `deinit` and explicit `detach()` are idempotent.
- Ensure `RuntimeSession` and `RuntimePTY` coordination does not permit calls through a stale Zig PTY pointer.
- Prefer internal initializer/ownership changes over preserving unsafe retain patterns.

3. **FFI lifetime hardening (`ffi.zig`):**
- Audit `SessionBox`/`PtyBox` retain-release rules under concurrent `disconnect`, `pty_write`, `pty_resize`, and `detach`.
- Ensure `smithers_core_disconnect` only closes API-level access while allowing outstanding PTY boxes to release final strong refs safely.
- Ensure `smithers_core_detach_pty` is strictly one-shot and cannot double-release session refs.
- Confirm all PTY operations acquire session safely and return deterministic closed/detached errors after teardown.

4. **Transport worker publication + concurrency fixes (`transport.zig`):**
- Subscription workers: publish to worker list only after thread/lifetime fields are initialized; preserve rollback correctness on append failure.
- PTY workers: apply the same publish-after-init rule and keep lock-order discipline (`transport mutex -> worker write mutex`) consistent.
- Re-check destroy/unsubscribe/detach paths to ensure no free occurs before thread handle initialization and join path is always valid.
- Strengthen write/resize vs detach/destroy discipline so worker memory cannot be freed while operation holds required lock/ownership.

5. **Test additions/updates:**
- Swift tests (`SmithersRuntimeTests.swift`): add/extend case where PTY handle outlives local `RuntimeSession` references and verify no UAF/crash (operations fail safely after disconnect/detach).
- Zig tests (`ffi.zig`, `transport.zig`, optionally `libsmithers/test/**`):
  - Deterministic race-focused tests for attach/write/resize/detach/destroy interleavings.
  - Assert worker publication invariants (thread initialized before visibility).
  - Assert disconnect safety with outstanding PTY wrappers.

6. **Regression pass + cleanup:**
- Run full required verification commands.
- Remove any temporary debug hooks; keep only durable assertions/tests/comments.

## Validation Strategy
- Required commands:
  - `cd libsmithers && zig build test --summary all`
  - `cd .. && swift test`
- Targeted checks during implementation:
  - `zig test libsmithers/src/core/ffi.zig` and `zig test libsmithers/src/core/transport.zig` for fast iteration.
  - Focused Swift runtime tests for PTY/session lifetime path.
- Acceptance mapping:
  - PTY-outlives-session Swift test present and passing.
  - Zig race-path tests present and passing.
  - `smithers_core_disconnect` safe with outstanding PTY wrappers.
  - No worker can be destroyed before thread handle init/publication.

## Known Risks
- **Cross-language lifetime coupling risk:** Swift ARC and Zig manual refs can diverge if detach/deinit ordering changes; mitigate with one-shot detach and explicit retain/release invariants.
- **Lock-order deadlock risk:** tightening mutex discipline can introduce deadlocks if order is inconsistent; mitigate with a single documented order and tests that exercise detach/write concurrency.
- **Flaky race tests risk:** stress-style tests can be nondeterministic; prefer deterministic fake transport gates/barriers where possible, with bounded stress loops only as supplemental coverage.
- **Behavioral regression risk:** stricter closed/detached error behavior may surface in callers; mitigate by preserving public API and validating existing runtime tests remain green.
